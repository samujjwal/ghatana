---

## Purpose

`@ghatana/canvas` is the hybrid renderer library for all Ghatana products that need interactive graph/diagram/flow canvas UI. It provides:

- **Custom renderer**: lightweight canvas engine using Web Components (Lit)
- **ReactFlow bridge**: `@xyflow/react` integration for flow diagrams
- **CRDT collaboration**: `yjs` for real-time multi-user editing
- **Topology tools**: DAG layout via `dagre`
- **Plugin system**: extensible node/edge renderers per product

## Source vs. Build Artifacts

**IMPORTANT:** There are two canvas directories in `platform/typescript/`:

| Path | Role | Status |
|------|------|--------|
| `platform/typescript/capabilities/canvas-core/` | ✅ **Canonical package** — registered in pnpm workspace. Source in `src/`. | Use this |
| `platform/typescript/canvas/` | ⚠️ **Legacy/WIP directory** — raw TypeScript source, no `package.json`. Not a workspace package. | Do not reference; migrate to canvas-core |

The `capabilities/canvas-core` `dist/` folder MUST NOT be committed. A `.gitignore` has been added to enforce this (BDY-2-3). Build outputs are generated by `pnpm build`.

---

## Governance

- **Breaking changes:** Require approval from both Data-Cloud and YAPPC leads
- **New consumers:** Update this library index and ensure any required module-state changes are reflected in `settings.gradle.kts`
- **API stability:** `./react`, `./types`, and `./hybrid` exports are stable. Others considered internal
- **Versioning:** Semantic versioning; minor versions for new features, patch for fixes
- **Build:** `pnpm --filter @ghatana/canvas build`
- **Test:** `pnpm --filter @ghatana/canvas test`
