## 1. Purpose

This document enumerates all **shared contracts** in the Ghatana monorepo: the event
envelope, AI governance contract, and observability naming conventions. Every
service and product that emits events, calls AI models, or emits metrics must
follow these contracts exactly.

---

## 2. Event Envelope Contract

All platform events use the canonical envelope defined in `platform/contracts`.

### 2.1 Wire Format (JSON)

```json
{
  "id":            "<uuid-v4>",
  "tenantId":      "<tenant-id>",
  "eventType":     "<domain>.<noun>.<verb>",
  "occurredAtMs":  1700000000000,
  "correlationId": "<uuid-v4>",
  "schemaVersion": "1",
  "source":        "<service-identifier>",
  "payload":       "<json-string or object>"
}
```

### 2.2 Field Specification

| Field | Type | Required | Constraints |
|-------|------|:---:|---|
| `id` | UUID v4 string | **Yes** | Must be globally unique; generated by the emitting service |
| `tenantId` | string | **Yes** | Must match `^[a-zA-Z0-9_-]{1,64}$`; never `"default-tenant"` in production |
| `eventType` | string | **Yes** | Format: `<domain>.<noun>.<verb>` (e.g., `auth.token.issued`, `data.entity.created`) |
| `occurredAtMs` | int64 | **Yes** | Epoch milliseconds (UTC); monotonically increasing within a session |
| `correlationId` | UUID v4 string | **Yes** | Must match the active request's `X-Correlation-ID`; propagated from inbound request |
| `schemaVersion` | string | **Yes** | Semver major only (e.g., `"1"`, `"2"`); increment on breaking field changes |
| `source` | string | **Yes** | Service identifier (e.g., `"auth-gateway"`, `"aep-server"`); kebab-case |
| `payload` | string or object | No | Domain-specific payload; must not contain raw PII in plaintext at the envelope level |

### 2.3 Registered Event Types

| Event Type | Source | Description |
|-----------|--------|-------------|
| `auth.token.issued` | auth-gateway | Access or refresh token minted |

| `compliance.gdpr.erasure_fulfilled` | data-cloud, aep | GDPR erasure request processed |

### 2.4 Adding a New Event Type

1. Add the event type to the registered list in this document.
2. Define the payload schema in `platform/contracts/schemas/events/<domain>/`.
3. Add a round-trip serialization test in `platform/contracts/src/test/java/**/*RoundTripTest.java`.
4. Emit the event using `platform:java:observability` helpers or the equivalent TypeScript `EventBus`.
5. Never emit cross-tenant events (the `tenantId` field must always match the active tenant).

## 3. AI Governance Contract

All AI model invocations must comply with the governance contract defined in
`docs/AI_GOVERNANCE_CONTRACTS.md`. Key requirements:

- Every LLM call must carry a `promptVersion` identifier.
- Responses must be logged at DEBUG level with correlation ID (no PII).
- Hallucination rate and latency must be measured against the registered eval suite.
- Model changes require an approved A/B test before becoming the default.

## 4. Observability Naming Conventions

### 4.1 Metric Naming

All metrics follow the format: `<product_or_platform>_<noun>_<unit>_<aggregation?>`

| Scope | Prefix | Examples |
|-------|--------|---------|
| Platform Java | `platform_` | `platform_db_query_duration_ms`, `platform_cache_hit_rate` |
| Auth Gateway | `auth_` | `auth_login_latency`, `auth_token_issued_total` |
| Data Cloud | `datacloud_` | `datacloud_entity_query_duration_ms`, `datacloud_event_append_total` |
| AEP | `aep_` | `aep_agent_task_duration_ms`, `aep_queue_depth` |
| YAPPC | `yappc_` | `yappc_phase_duration_ms`, `yappc_artifact_generated_total` |
| Audio-Video | `av_` | `av_session_duration_ms`, `av_stt_latency_ms` |

**Rules:**
- Use snake_case.
- Suffix with `_total` for counters, `_ms` for millisecond histograms.
- Include `tenantId` as a label only for low-cardinality tenant sets;
  otherwise use a tenant-count rollup to avoid high-cardinality label explosion.

### 4.2 Trace Span Naming

Format: `<service>.<operation>` (e.g., `auth-gateway.login`, `data-cloud.entity.query`).

**Required span attributes:**
- `tenant.id` — current tenant
- `correlation.id` — request correlation ID
- `service.name` — emitting service
- `service.version` — deployed version

### 4.3 Log Entry Schema

All structured log entries must be valid JSON with at minimum:

```json
{
  "timestamp":     "2026-04-24T12:00:00.000Z",
  "level":         "INFO|WARN|ERROR|DEBUG",
  "service":       "auth-gateway",
  "correlationId": "uuid",
  "tenantId":      "tenant",
  "message":       "Human-readable summary"
}
```

**Forbidden:** raw PII in log entries (names, emails, card numbers, health data).
Use `[REDACTED]` placeholders for sensitive fields.

## 5. Shared TypeScript Contracts

### 5.1 Event Bus (`@ghatana/events`)

All TypeScript packages that emit platform events must use `@ghatana/events`:

```typescript
import { EventBus } from '@ghatana/events';

EventBus.emit({
  id: crypto.randomUUID(),
  tenantId: getTenantId(),
  eventType: 'data.entity.created',
  occurredAtMs: Date.now(),
  correlationId: getCorrelationId(),
  schemaVersion: '1',
  source: 'data-cloud-client',
  payload: { entityId, collectionName }
});
```

**Forbidden:** raw `fetch`/`axios` calls to the event ingestion endpoint from
product code — always use `@ghatana/events` to ensure envelope compliance.

### 5.2 State Atoms (`@ghatana/state`)

Platform-level state (auth session, tenant identity, notifications) must use
atoms from `@ghatana/state`. Product packages wrap these atoms but must not
re-declare platform-level state.

**Canonical atoms:**
- `authAtom` — authenticated user session
- `tenantAtom` — active tenant identity
- `notificationAtom` — platform-level notification queue
