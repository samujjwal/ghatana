# TutorPutor Capability Traceability Matrix

This is the canonical capability-to-implementation trace for TutorPutor. Every product promise must have an owning contract, backend owner, frontend surface, persistence owner, and verification reference before it can be described as implemented.

Canonical runtime model: TutorPutor is a deliberately composed hybrid with one primary TypeScript platform runtime. `services/tutorputor-platform` owns product APIs, policy, persistence, telemetry ingest, analytics, compliance, LTI, credentials, and orchestration. Java/ActiveJ agent libraries and content-generation services are worker runtimes behind typed platform boundaries. `apps/api-gateway` is a thin edge gateway, not an alternate product backend.

## Enforcement

`scripts/validate-product-traceability.mjs` validates this matrix in CI. It fails when a required capability row is missing, any owner column is empty, any referenced file is missing, or the public OpenAPI route map lacks a canonical backend owner and typed client.
