# Tutorputor – Coding Guidelines

## 1. Scope

These guidelines apply to `products/tutorputor/*`.

## 2. Design Principles

- Keep contracts in `contracts/` as the source of truth for APIs and data models.
- Implement orchestration and stateful flows in `services/`.
- Keep `apps/` focused on presentation and client interactions.

This document is self-contained and defines how to structure code in Tutorputor.
