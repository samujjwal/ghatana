# Tutorputor Web App – Coding Guidelines

## 1. Scope

These guidelines apply to `products/tutorputor/apps/tutorputor-web`.

## 2. Design Principles

- Use function components and hooks.
- Keep components presentational; move data fetching and mutations to hooks/services.
- Model client state in a way that mirrors Tutorputor contracts and backend APIs.

This document is self-contained and defines how to structure code in the Tutorputor Web app.
