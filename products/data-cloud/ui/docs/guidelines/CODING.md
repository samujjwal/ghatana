# CES UI – Coding Guidelines

## 1. Scope

These guidelines apply to `products/collection-entity-system/ui`.

## 2. Design Principles

- Keep components presentational; delegate data fetching and mutations to hooks/services.
- Use typed models that align with CES API/domain types.
- Reuse shared UI components and styles where possible.

This document is self-contained and defines how to structure code in the CES UI module.
