## How to Read This Plan

**Every task** lists:

- **ID** — Unique, stable reference (PHASE.TRACK.SEQUENCE)
- **Title** — What gets done
- **Module** — Exact Gradle/npm module or file path touched
- **Depends On** — Predecessor task IDs (empty = can start immediately)
- **Acceptance Criteria** — What "done" looks like, testably
- **Ghatana Compliance** — Which copilot-instructions.md rules the task must satisfy

**Phase structure:**

- **Phase 0** — Build health and quality gates (prerequisite for all other work)
- **Phase 1** — Trust foundation (durable persistence, auth, tenant isolation, API contracts)
- **Phase 2** — Enterprise readiness (real governance, RBAC, observability, plugin lifecycle)
- **Phase 3** — Competitive differentiation (context layer API, federated query, voice, features)
- **Phase 4** — Future-ready (autonomous ops, sovereign mode, data products, embeddings)

**Parallelism guidance:** Tasks within the same phase that share no `Depends On` can run in parallel. Cross-phase dependencies are explicit.
