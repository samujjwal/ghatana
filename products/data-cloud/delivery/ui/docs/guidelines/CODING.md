# Data Cloud UI – Coding Guidelines

## 1. Scope

These guidelines apply to `products/data-cloud/delivery/ui`.

## 2. Design Principles

- Keep components presentational; delegate data fetching and mutations to hooks/services.
- Use typed models that align with Data Cloud API/domain types.
- Reuse shared UI components and styles where possible.

This document is self-contained and defines how to structure code in the Data Cloud UI module.
