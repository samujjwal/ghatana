# Claim Traceability Matrix

Version: 1.0.1  
Status: Compact cross-reference for factual and legal claims used in active Project Siddhanta documents  
Last revised: March 10, 2026

---

## Purpose

This matrix provides a compact bridge between:

- factual source register IDs (`ASR-*`),
- legal claim IDs (`LCA-*`),
- source URLs,
- and the active documents that rely on those claims.

Use this when reviewing customer-facing requirements, investment memos, or regulator-facing summaries.

**Namespace Note:** This matrix tracks the numeric legal-claim identifiers from [docs/Legal_Claim_Citation_Appendix.md](/home/samujjwal/Developments/finance/docs/Legal_Claim_Citation_Appendix.md) alongside factual source-register IDs. It is intentionally separate from the semantic epic/control code namespace maintained in [epics/COMPLIANCE_CODE_REGISTRY.md](/home/samujjwal/Developments/finance/epics/COMPLIANCE_CODE_REGISTRY.md).

## Usage Rules

1. If a row has both `LCA-*` and `ASR-*`, the legal and factual parts should both be checked before externalizing the claim.
2. If a row has `n/a (pending legal)`, do not market it as a clause-verified legal requirement.
3. If a row has `n/a (factual)`, it is a date-sensitive metric rather than a legal obligation.
4. This matrix is a navigation aid; the controlling detail remains in [Legal_Claim_Citation_Appendix.md](./Legal_Claim_Citation_Appendix.md) and [Authoritative_Source_Register.md](./Authoritative_Source_Register.md).
