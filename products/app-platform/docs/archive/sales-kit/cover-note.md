# Sales Kit Cover Note

Version: 1.0  
Status: Routing note for selecting the right customer-facing insert by buyer type  
Last revised: March 1, 2026

Canonical sales-kit index: [README.md](./README.md)

---

## Purpose

This note tells a sender which insert to use for which audience.

Use it before sending any proposal excerpt so the recipient gets the shortest, most relevant version first.

---

## Which File To Send

### If the recipient is broad or mixed

Send:

- [One_Page_Proposal_Insert.md](../One_Page_Proposal_Insert.md)

Optional support:

- [email_blurbs.md](./email_blurbs.md)

Use when:

- the audience is mixed,
- the first conversation is exploratory,
- the recipient does not fit a single buyer type yet.

### If the recipient is a broker or DP

Send:

- [Broker_Proposal_Insert.md](../Broker_Proposal_Insert.md)

Optional support:

- [email_blurbs.md](./email_blurbs.md)

Use when:

- the buyer cares most about broker controls,
- the conversation is about post-trade operations, margin, reporting, or auditability.

### If the recipient is a merchant banker

Send:

- [Merchant_Banker_Proposal_Insert.md](../Merchant_Banker_Proposal_Insert.md)

Optional support:

- [email_blurbs.md](./email_blurbs.md)

Use when:

- the buyer cares about issue execution, prospectus workflow, due diligence, or issue-close reporting.

### If the recipient is a listed issuer

Send:

- [Issuer_Proposal_Insert.md](../Issuer_Proposal_Insert.md)

Optional support:

- [email_blurbs.md](./email_blurbs.md)

Use when:

- the buyer cares about disclosure timing, filing calendars, governance packs, or corporate actions.

## Which Tailored Profile To Use

### For a mid-market full-service broker

Send:

- [Mid_Market_Full_Service_Broker_Proposal_Insert.md](./Mid_Market_Full_Service_Broker_Proposal_Insert.md)

Use when:

- the prospect is a realistic Nepal broker profile with DP workflows, margin-service ambition, and manual reporting pressure.

### For a mid-sized merchant banker

Send:

- [Mid_Sized_Merchant_Banker_Proposal_Insert.md](./Mid_Sized_Merchant_Banker_Proposal_Insert.md)

Use when:

- the prospect is a realistic merchant banker profile with recurring filing-pack, allotment, and issue-close reporting pressure.

### For a listed issuer with frequent corporate actions

Send:

- [Listed_Issuer_Frequent_Corporate_Actions_Proposal_Insert.md](./Listed_Issuer_Frequent_Corporate_Actions_Proposal_Insert.md)

Use when:

- the prospect is a listed company with repeated book-closure, dividend, rights, or board-notice activity.

## Escalation Rule

If the recipient challenges a claim or asks for proof, move to:

1. [Customer_Facing_Claims_Pack.md](../Customer_Facing_Claims_Pack.md)
2. [Legal_Claim_Citation_Appendix.md](../Legal_Claim_Citation_Appendix.md)
3. [Authoritative_Source_Register.md](../Authoritative_Source_Register.md)

Do not improvise new legal or factual claims in outbound material without updating those source documents first.

Before writing a named version:

1. capture only verified context using [Prospect_Context_Capture_Checklist.md](./Prospect_Context_Capture_Checklist.md)
2. customize from [Named_Prospect_Insert_Template.md](./Named_Prospect_Insert_Template.md)
3. keep unsupported details as assumptions or leave them blank
