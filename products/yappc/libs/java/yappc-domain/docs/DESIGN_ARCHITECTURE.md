# YAPPC Domain – Design & Architecture

## 1. Purpose

The **`libs/java/yappc-domain`** library provides domain models and logic for the YAPPC (Yet Another Project/Process/Case) domain within Ghatana.

## 2. Responsibilities

- Model YAPPC entities, value objects, and workflows.
- Provide domain behavior independent of transport and persistence.

## 3. Architectural Position

- Domain-focused library under `/libs/java`.
- Used by YAPPC-related services as their canonical domain layer.

This document is self-contained and summarizes the role and architecture of the YAPPC Domain library.
