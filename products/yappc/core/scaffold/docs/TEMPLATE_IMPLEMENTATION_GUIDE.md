# Template Implementation Guide

This guide explains how to create and maintain Handlebars templates for YAPPC scaffold packs.

---

## Template Basics

### File Naming
- Templates use `.hbs` extension (Handlebars)
- Output filename is template name without `.hbs`
- Example: `main.go.hbs` → `main.go`

### Variable Syntax
```handlebars
{{variableName}}           # Simple variable
{{pascalCase projectName}} # With helper
{{#if condition}}...{{/if}} # Conditional
{{#each items}}...{{/each}} # Loop
```

### Available Helpers
- `lowercase` - Convert to lowercase
- `uppercase` - Convert to UPPERCASE
- `capitalize` - Capitalize first letter
- `camelCase` - Convert to camelCase
- `pascalCase` - Convert to PascalCase
- `snakeCase` - Convert to snake_case
- `kebabCase` - Convert to kebab-case
- `uuid` - Generate UUID
- `date` - Current date (YYYY-MM-DD)
- `now` - Current timestamp (ISO 8601)

## Template Structure by Language

### Go Templates
**Required Variables:**
- `projectName` - Project name
- `modulePath` - Go module path (e.g., github.com/user/project)
- `goVersion` - Go version (e.g., 1.21)

**Optional Variables:**
- `description` - Project description
- `author` - Author name
- `enableSwagger` - Enable Swagger/OpenAPI
- `enableCors` - Enable CORS middleware
- `enableMetrics` - Enable Prometheus metrics

**Example:**
```handlebars
package main

import (
    "{{modulePath}}/internal/config"
    {{#if enableMetrics}}
    "{{modulePath}}/internal/metrics"
    {{/if}}
)

func main() {
    // {{description}}
    {{#if enableMetrics}}
    metrics.Init()
    {{/if}}
}
```

### Python Templates
**Required Variables:**
- `projectName` - Project name
- `pythonVersion` - Python version (e.g., >=3.11)

**Optional Variables:**
- `description` - Project description
- `buildTool` - Build tool (uv, poetry, pip)
- `enableSwagger` - Enable FastAPI docs
- `enableCors` - Enable CORS
- `enableMetrics` - Enable Prometheus

**Example:**
```handlebars
"""
{{projectName}} - {{description}}
"""
from fastapi import FastAPI

app = FastAPI(
    title="{{projectName}}",
    {{#if enableSwagger}}
    docs_url="/docs",
    {{else}}
    docs_url=None,
    {{/if}}
)
```

### Rust Templates
**Required Variables:**
- `projectName` - Project name
- `rustEdition` - Rust edition (2021)

**Optional Variables:**
- `description` - Project description
- `enableCors` - Enable CORS
- `enableMetrics` - Enable metrics

**Example:**
```handlebars
use actix_web::{App, HttpServer};
{{#if enableCors}}
use actix_cors::Cors;
{{/if}}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            {{#if enableCors}}
            .wrap(Cors::default())
            {{/if}}
    })
    .bind("0.0.0.0:8080")?
    .run()
    .await
}
```

### .NET Templates
**Required Variables:**
- `projectName` - Project name
- `targetFramework` - Target framework (net8.0)

**Optional Variables:**
- `description` - Project description
- `enableNullable` - Enable nullable reference types
- `enableSwagger` - Enable Swagger
- `enableCors` - Enable CORS

**Example:**
```handlebars
<Project Sdk="Microsoft.NET.Sdk.Web">
  <PropertyGroup>
    <TargetFramework>{{targetFramework}}</TargetFramework>
    {{#if enableNullable}}
    <Nullable>enable</Nullable>
    {{/if}}
  </PropertyGroup>
</Project>
```

## Best Practices

### 1. Use Defaults
Always provide sensible defaults in pack.json:
```json
{
  "defaults": {
    "goVersion": "1.21",
    "enableSwagger": "true",
    "enableCors": "true"
  }
}
```

### 2. Conditional Imports
Only import what's needed:
```handlebars
{{#if enableMetrics}}
import "github.com/prometheus/client_golang/prometheus"
{{/if}}
```

### 3. Comments
Add helpful comments:
```handlebars
// {{projectName}} - {{description}}
// Generated by YAPPC on {{date}}
```

### 4. Formatting
Maintain proper indentation:
```handlebars
func main() {
    {{#if enableMetrics}}
    // Initialize metrics
    metrics.Init()
    {{/if}}
}
```

### 5. Error Handling
Include error handling patterns:
```handlebars
if err := server.Start(); err != nil {
    log.Fatalf("Failed to start: %v", err)
}
```

## Testing Templates

### Manual Testing
```bash
yappc create my-project go-service-gin \
  --var projectName=my-service \
  --var modulePath=github.com/user/my-service
```

### Golden Tests
Create expected output files and compare:
```java
@Test
void testGoServiceGinTemplate() {
    Map<String, String> vars = Map.of(
        "projectName", "test-service",
        "modulePath", "github.com/test/service"
    );
    
    String result = templateEngine.render("main.go.hbs", vars);
    String expected = Files.readString(Path.of("golden/main.go"));
    
    assertEquals(expected, result);
}
```

## Common Patterns

### Configuration Files
```handlebars
# {{projectName}} Configuration
# Generated on {{date}}

app:
  name: {{projectName}}
  version: 1.0.0
  {{#if description}}
  description: {{description}}
  {{/if}}
```

### Dockerfiles
```handlebars
FROM golang:{{goVersion}}-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o {{projectName}} .

FROM alpine:latest
COPY --from=builder /app/{{projectName}} /usr/local/bin/
CMD ["{{projectName}}"]
```

### Makefiles
```handlebars
.PHONY: build test clean

build:
	go build -o bin/{{projectName}} .

test:
	go test -v ./...

clean:
	rm -rf bin/
```

## Troubleshooting

### Variable Not Found
Ensure variable is in pack.json requiredVariables or optionalVariables.

### Syntax Errors
Check Handlebars syntax - common issues:
- Missing closing tags: `{{#if}}...{{/if}}`
- Typos in helper names
- Incorrect nesting

### Whitespace Issues
Use `~` to trim whitespace:
```handlebars
{{~#if condition~}}
content
{{~/if~}}
```
