# ESLint Configuration - Excluded Patterns

This document explains what files and directories are excluded from ESLint linting.

## Build & Generated Artifacts

- `dist/`, `build/`, `coverage/` - Build output directories
- `.turbo/`, `.next/`, `.parcel-cache/`, `.cache/` - Tool caches
- `test-results/`, `e2e-debug/`, `playwright-report/`, `verbose/` - Test artifacts

## Compiled Files

- `**/*.js`, `**/*.cjs` - Compiled JavaScript (TypeScript source only)
- `**/*.d.ts` - TypeScript declaration files
- `**/*.config.js`, `**/*.setup.js` - Config files that are compiled
- `**/*.test.js`, `**/*.spec.js` - Compiled test files

## Test Files

- `e2e/**` - End-to-end tests (Playwright)
- `**/*.spec.ts`, `**/*.spec.tsx` - Spec test files
- `debug-*.ts` - Debug test files
- `**/__tests__/**/*.js` - Compiled test directories

## Generated Code

- `**/generated/**`, `**/_generated/**` - Codegen output directories
- `**/*.generated.ts`, `**/*.generated.tsx` - Generated files
- `libs/graphql/src/generated/**` - GraphQL codegen output
- `**/graphql-codegen/**`, `**/codegen/**` - Codegen tool directories

## External/Vendored Code

- `**/vendor/**` - Vendored third-party code
- `**/third-party/**` - Third-party integrations
- `**/external/**` - External code

## Demo/Example Code

- `examples/**` - Example code
- `.storybook/**` - Storybook configuration
- `packages/**` - Package root configs
- `docs-site/**` - Documentation sites

## Config Files

- `playwright.config.ts` - Playwright configuration
- `sentry*.config.ts` - Sentry configuration
- `simple.test.ts` - Simple test files
- `.eslintrc.cjs` - ESLint configuration

## Tool Output

- `**/.next/**` - Next.js build output
- `**/out/**` - Build output
- `**/.vercel/**` - Vercel deployment config
- `.dependency-cruiser.js` - Dependency cruiser config
- `true/**` - Misc output

## Why These Exclusions?

1. **Generated Code**: Cannot be manually fixed - changes are overwritten on regeneration
2. **Compiled Files**: Linting source `.ts` files is sufficient; compiled `.js` adds redundancy
3. **Build Artifacts**: Not source code - temporary files
4. **External Code**: Not owned by project - vendor code shouldn't be linted
5. **Examples/Docs**: Not part of core library - different standards

## How to Lint

```bash
# Standard lint
pnpm lint

# Fix auto-fixable issues
pnpm lint:fix

# Fast lint (apps and libs only)
pnpm lint:fast
```

## If You Need to Lint Something Excluded

You can override ignores in ESLint:

```bash
eslint ./some-file.ts --no-ignore
```

Or modify the patterns in:
- `eslint.config.mjs` - ESLint flat config ignores
- `package.json` - Script ignore patterns
