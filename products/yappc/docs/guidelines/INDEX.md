# YAPPC – Guidelines

This directory groups **YAPPC-specific guidelines**, layered on top of the global platform rules in `docs/global/`.

## Planned documents

- `CODING.md` – Java/TypeScript coding patterns used in YAPPC (backend + app-creator).
- `TESTING.md` – Testing strategy (backend, frontend, E2E) and coverage targets.
- `QUALITY.md` – Non-functional goals (performance, reliability, UX quality).
- `SECURITY.md` – Security posture for YAPPC services and UI.
- `BUILD.md` – Build and CI details for backend and app-creator packages.
- `TOOLS_LIBS.md` – Libraries and tools used by YAPPC (ActiveJ, React, Vite, StateManager, etc.).
- `CHANGE_MANAGEMENT_DEPRECATION.md` – How to evolve YAPPC APIs/components with deprecations.

Each guideline file should make explicit which global rules it inherits and what additional YAPPC-specific constraints apply.
