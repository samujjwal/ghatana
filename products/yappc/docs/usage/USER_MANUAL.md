# YAPPC – User Manual (For Developers)

## 1. Who This Is For

This manual is for developers and teams integrating with YAPPC backend services or working on the app-creator.

## 2. Backend Usage

- Use documented HTTP APIs for requirements, refactoring, and knowledge graph operations.
- Follow platform HTTP and observability conventions.

Typical backend usage patterns:

- Send requests to generate or refine requirements and designs.
- Use refactoring and knowledge graph endpoints to analyze and evolve existing systems.

## 3. App-Creator Usage

- Use the app-creator to design and manage applications visually.
- Rely on shared components and state patterns to keep UI consistent with platform standards.

Typical app-creator flows:

- Create or open an app in the canvas.
- Use the component palette and configuration panels to design pages.
- Save and later reload applications through the backend integration.

This manual is self-contained and gives a high-level view of how YAPPC is used.
