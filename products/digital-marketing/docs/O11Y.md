# DMOS Observability Guide

## Overview

DMOS uses OpenTelemetry for distributed tracing, metrics, and logging. All telemetry is sent to the configured OTLP endpoint.

## Tracing

### Enable Tracing

Set the environment variable:
```bash
export OTEL_EXPORTER_OTLP_ENDPOINT="http://jaeger:4317"
export OTEL_SERVICE_NAME="dmos-api"
```

### View Traces

Traces are available in Jaeger at `http://jaeger:16686`

## Metrics

### Key Metrics

- `dmos_api_requests_total`: Total API requests
- `dmos_api_request_duration_seconds`: Request duration
- `dmos_connector_executions_total`: Connector executions
- `dmos_connector_execution_duration_seconds`: Connector execution duration
- `dmos_approval_latency_seconds`: Approval latency
- `dmos_strategy_generation_duration_seconds`: Strategy generation duration

### View Metrics

Metrics are available in Prometheus at `http://prometheus:9090`

## Logging

### Log Levels

- ERROR: Errors requiring immediate attention
- WARN: Warning conditions
- INFO: Informational messages
- DEBUG: Debug messages

### Structured Logging

Logs include:
- Timestamp
- Log level
- Tenant ID
- Workspace ID
- Request ID
- Message

## Dashboards

### Pre-configured Dashboards

1. **API Performance Dashboard**: Request rate, latency, error rate
2. **Connector Dashboard**: Connector execution metrics
3. **Approval Dashboard**: Approval latency and throughput
4. **Strategy Generation Dashboard**: AI generation metrics
5. **Database Dashboard**: Connection pool, query performance

### Access Dashboards

Grafana is available at `http://grafana:3000`

## Alerts

### Configured Alerts

1. **High Error Rate**: > 5% error rate for 5 minutes
2. **High Latency**: P95 latency > 2 seconds for 5 minutes
3. **Connector Failures**: > 10% connector failure rate
4. **Database Connection Pool Exhaustion**: Pool usage > 90%
5. **Approval Backlog**: > 100 pending approvals

### View Alerts

Alertmanager is available at `http://alertmanager:9093`
