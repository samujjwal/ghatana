PROBLEM STATEMENT
================================================================================

Original Issue:
"Currently I do not see data, page name, domain names etc"

Root Cause Analysis:
1. Content script wasn't including status field in collected data
2. GET_USAGE_SUMMARY handler delegated to plugin instead of using storage
3. Type errors prevented proper compilation
4. Dashboard had incomplete data source

================================================================================
DATA FLOW (AFTER FIX)
================================================================================

1. User visits website (e.g., https://github.com)
   ↓
2. Content Script:
   - Extracts: URL, domain (github.com), title, timestamp
   - Sets: status = 'allowed', sessionDuration = 0
   - Sends: PAGE_USAGE_TRACKED message to background
   ↓
3. Background Service Worker:
   - Receives PAGE_USAGE_TRACKED
   - Stores in: browser.storage.local["guardian-usage"]
   - Updates state counters
   ↓
4. Every 30 seconds:
   - Content script updates sessionDuration
   - Resends PAGE_USAGE_TRACKED with new duration
   ↓
5. Dashboard requests GET_ANALYTICS:
   - Controller reads guardian-usage storage
   - Groups all visits by domain
   - Calculates:
     ✓ Visit count per domain
     ✓ Total time per domain
     ✓ Blocked count
     ✓ Last visited timestamp
   - Returns topDomains array
   ↓
6. Dashboard Renders:
   ✓ Domain names (google.com, github.com)
   ✓ Page titles ("Google", "GitHub")
   ✓ Full URLs (https://...)
   ✓ Visit counts (8, 5, 3)
   ✓ Time spent (2m 15s)
   ✓ Status badges (✅ Allowed, 🚫 Blocked)
   ✓ Timestamps (relative: "5 minutes ago")
   ✓ Activity log (recent 50 pages)
