# Guardian – Parent Mobile – Coding Guidelines

## 1. Scope

These guidelines apply to `@guardian/parent-mobile`.

## 2. Component & State Design

- Use function components and hooks; organize screens by feature.
- Use React Navigation for navigation and React Query for server state.
- Keep network and validation logic in data modules, not in components.

## 3. Notifications & Permissions

- Handle notification and background permissions explicitly and with clear UX.

This document is self-contained and defines how to structure code in `@guardian/parent-mobile`.
