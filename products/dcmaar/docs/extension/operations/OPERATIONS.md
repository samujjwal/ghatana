# DCMaar Browser Extension – Operations Guide

## 1. Overview

The browser extension is distributed via browser stores or internal channels. Operations focus on rollout, updates, and monitoring behavior at scale.

## 2. Distribution & Updates

- Publish to allowed browser stores or distribute signed packages internally.
- Use extension versioning and staged rollout where possible.

## 3. Configuration

- Manage domain allowlists, redaction rules, and connection settings centrally when supported.
- Ensure configuration changes are tested on staging profiles before production.

## 4. Monitoring

- Monitor aggregate metrics from agents/server related to browser events.
- Watch error logs for extension-side failures or configuration drift.

This guide is self-contained and documents operational considerations for the DCMaar Browser Extension.
