# DCMaar Runtime Config Types – Coding Guidelines

## 1. Scope

These guidelines apply to the `runtime-config-types` crate.

## 2. Generation & Manual Edits

- Treat this crate primarily as generated code from a shared schema.
- Avoid hand-editing generated types; instead, change the source schema and regenerate.

## 3. Type Design

- Keep configuration types expressive and aligned with actual runtime needs.

This document is self-contained and defines how to work with `runtime-config-types`.
