# DCMaar – Plugin Abstractions – Coding Guidelines

## 1. Scope

These guidelines apply to `@dcmaar/plugin-abstractions`.

## 2. API Design

- Keep plugin interfaces small, focused, and stable.
- Use `@dcmaar/types` for shared domain models.

## 3. Backwards Compatibility

- Avoid breaking existing plugin code; prefer new interfaces or feature flags.

This document is self-contained and defines how to structure code in `@dcmaar/plugin-abstractions`.
