# DCMaar – Shared UI Charts – Coding Guidelines

## 1. Scope

These guidelines apply to `@dcmaar/shared-ui-charts`.

## 2. Component Design

- Keep chart components generic and data-agnostic.
- Accept data and configuration via props; avoid hardcoding metrics or domains.

## 3. Dependencies

- Use `recharts` and `@dcmaar/shared-ui-tailwind` according to existing patterns.

This document is self-contained and defines how to structure code in `@dcmaar/shared-ui-charts`.
