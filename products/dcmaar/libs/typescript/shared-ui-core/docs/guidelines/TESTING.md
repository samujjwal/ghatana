# DCMaar – Shared UI Core – Testing Guidelines

## 1. Goals

- Ensure shared UI tokens, types, hooks, and utilities behave consistently across consumers.

## 2. Tests

- Use TypeScript type-checking and minimal runtime tests where appropriate.
- Validate that tokens and hooks behave correctly under typical usage patterns.

This document is self-contained and explains how to test `@dcmaar/shared-ui-core`.
