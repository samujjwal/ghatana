# Aura Engineering Sprint Plan (First 6 Months)

## Assumptions
- 2-week sprints
- Team: 1 PM, 1 designer, 3 full-stack engineers, 1 ML engineer, 1 QA/shared automation
- Initial focus: beauty-first MVP

## Month 1
### Sprint 1
- Finalize product scope and user journeys
- Stand up monorepo, CI/CD, environments
- Define domain model and Prisma schema
- Implement auth and profile onboarding skeleton

### Sprint 2
- Build catalog ingestion MVP
- Add product, brand, ingredient entities
- Create basic feed API
- Create wireframes and design system foundation

## Month 2
### Sprint 3
- Implement product detail page
- Build save/bookmark flows
- Add consent center v1
- Add data export request flow
- Add ingestion jobs and admin diagnostics

### Sprint 4
- Implement ingredient analyzer v1
- Add rules engine for allergen and ethical filtering
- Build recommendation explanation payload shape
- Add unit and integration tests

## Month 3
### Sprint 5
- Implement shade-matching v1
- Add recommendation query API
- Build personalized recommendation cards in UI
- Instrument analytics events

### Sprint 6
- Improve ranking heuristics
- Add compare products screen
- Add feedback capture: view, click, save, dismiss
- Add post-use outcome event contract: shade mismatch, reaction, return
- Run internal alpha

## Month 4
### Sprint 7
- Add review ingestion and sentiment pipeline
- Expand product enrichment jobs
- Improve profile editing and inferred-attribute display
- Harden observability and error budgets

### Sprint 8
- Add assistant query flow
- Implement search + guided recommendation prompts
- Refine explanations and confidence display
- Add user-facing outcome reporting in product detail and saved items
- Begin external beta prep

## Month 5
### Sprint 9
- Beta launch
- Add experimentation framework
- Tune ranking using interaction data
- Add moderation tools for review/community ingestion

### Sprint 10
- Affiliate link instrumentation
- Conversion funnel dashboards
- Recommendation quality and safety outcome reviews
- Privacy and trust UX pass

## Month 6
### Sprint 11
- Optimize ingestion coverage and freshness
- Improve recommendation latency
- Add backlog items from beta learning
- Start premium packaging analysis

### Sprint 12
- Production hardening
- QA sweep and regression automation
- Launch readiness review
- Plan phase-2 features
