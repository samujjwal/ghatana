# Platform Contracts — Generated Artifact Policy

**Status:** Active  
**Last Updated:** 2026-04-23  
**Owner:** Platform Architecture Team  
**Slack:** #platform-contracts  

---

## 1. Purpose

This document defines which generated outputs in `platform/contracts` are source-of-truth
(must be version-controlled) and which are derived build outputs (must be excluded from
version control). It also defines the validation gates that enforce this policy.

---

## 2. What Lives in `platform/contracts`

`platform/contracts` is the canonical home for all **cross-service contract definitions**.
Every schema, protocol, or API that must be stable across product boundaries lives here.

| Artifact Type | Location | Source-of-Truth? | Notes |
|--------------|----------|-----------------|-------|
| Protobuf definitions (`.proto`) | `com/ghatana/**/*.proto` | **Yes — commit** | Hand-authored schemas; version-controlled as source |
| OpenAPI specifications (`.yaml`) | `openapi/*.yaml` | **Yes — commit** | Hand-authored API contracts; reviewed via PR |
| `buf.yaml` / `buf.gen.yaml` | Root | **Yes — commit** | Toolchain config for proto compilation |
| Build config (`build.gradle.kts`, `build-docs.gradle`) | Root | **Yes — commit** | Build instructions |

---

## 3. Generated Outputs — Exclude from Version Control

The following artifacts are produced by the build from the hand-authored sources above.
They must **never be committed** to version control. They are regenerated on every build.

| Generated Artifact | Produced By | Location | Excluded via `.gitignore`? |
|-------------------|------------|---------|---------------------------|
| Java proto stubs (`*.java`) | `protoc` / `buf` | `build/generated/source/proto/main/java/` | Yes |
| Proto descriptor sets (`*.bin`, `*.pb`) | `buf build` | `build/descriptors/` | Yes |
| OpenAPI-generated client/server stubs | `openapi-generator` (if used) | `build/generated-sources/` | Yes |
| Compiled classes (`*.class`) | `javac` | `build/classes/` | Yes (Gradle default) |
| JAR artifacts | `jar` task | `build/libs/` | Yes (Gradle default) |
| Jacoco coverage reports | `jacocoTestReport` | `build/reports/` | Yes |
| Checkstyle / PMD reports | static analysis tasks | `build/reports/` | Yes |

### Verification

The contract compatibility CI gate (`contractCompatibilityGate` Gradle task) validates that:

1. Proto files compile cleanly with `buf lint`
2. OpenAPI specs pass schema validation
3. No breaking changes are introduced (checked by `buf breaking`)
4. Consumer-driven contract tests pass for all registered consumers

## 4. Adding a New Contract

When adding a new cross-service contract to `platform/contracts`:

### 4.1 Protobuf Contract

1. Create `com/ghatana/<domain>/<name>.proto` with proper package and `java_package` options.
2. Ensure `buf.yaml` covers the new file path.
3. Add a consumer-driven contract test in `src/test/java/` named `*ContractTest.java`.
4. Do **not** commit the generated Java stubs — they are regenerated by Gradle.
5. Update this document's inventory table if a new domain is introduced.

### 4.2 OpenAPI Contract

1. Create `openapi/<service-name>.yaml` conforming to OpenAPI 3.x.
2. Register the spec in `build.gradle.kts` `copyProductOpenApiSpecs` task if it comes from a
   product root.
3. Add a contract compatibility test in `src/test/java/` named `*CompatibilityTest.java`.
4. Do **not** commit generated client stubs.

### 4.3 Event Schema Contract

1. Define the event envelope in the canonical format from
   `docs/architecture/PROPAGATION_CONTRACTS.md`.
2. If the event uses Protobuf, place the `.proto` in `com/ghatana/events/`.
3. If the event uses JSON, add a JSON Schema file under `schemas/events/`.
4. Add a round-trip test named `*RoundTripTest.java` validating serialization and
   deserialization correctness.

## 5. Breaking Change Policy

Breaking changes to contracts are subject to a mandatory review process:

| Change Type | Review Required | Deprecation Period |
|------------|----------------|-------------------|
| Add optional field (proto) | PR review | None |
| Add optional field (OpenAPI) | PR review | None |
| Remove field / endpoint | Architecture review + ADR | Minimum 1 release cycle |
| Rename field | Architecture review + ADR | Minimum 1 release cycle |
| Change field type | Architecture review + ADR | Minimum 1 release cycle |
| Change required/optional semantics | Architecture review | Minimum 1 release cycle |

`buf breaking` enforces proto backward-compatibility on every PR. OpenAPI compatibility is
enforced by the `contractCompatibilityGate` Gradle task.

---

## 6. `.gitignore` Rules for `platform/contracts`

The following pattern must be present in `platform/contracts/.gitignore` (or root `.gitignore`):

```gitignore
# Generated protobuf Java stubs
build/generated/source/proto/

# Generated descriptor sets
build/descriptors/

# Standard Gradle output
build/classes/
build/libs/
build/reports/
build/tmp/

# OpenAPI generated stubs (if applicable)
build/generated-sources/openapi/
```

---

## 7. CI Enforcement

| Gate | Gradle Task | Trigger |
|------|------------|---------|
| Proto lint | `buf lint` | Every PR |
| Proto breaking change check | `buf breaking` | Every PR against main |
| OpenAPI schema validation | `contractCompatibilityTest` | Every PR |
| Consumer contract tests | `contractCompatibilityGate` | Every PR + nightly |
| Generated file leak detection | `check-generated-files.sh` | Every PR |
