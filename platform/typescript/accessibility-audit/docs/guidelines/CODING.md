# Shared TypeScript Accessibility Audit – Coding Guidelines

## 1. Scope

These guidelines apply to `libs/typescript/accessibility-audit`.

## 2. Design Principles

- Keep accessibility helpers generic and reusable.
- Avoid coupling audit logic to specific UI frameworks beyond React where possible.
- Follow WCAG and internal accessibility standards when designing checks.

This document is self-contained and defines how to structure code in the Shared TypeScript Accessibility Audit library.
