# Contributing to Ghatana

## Development Setup

1. **Clone** the repository and ensure prerequisites are met:
   - Java 21 (via SDKMAN or similar)
   - Node.js 20+ with `pnpm` (`npm install -g pnpm`)
2. **Build**: `./gradlew clean build`
3. **Test**: `./gradlew test`
4. **Format**: `./gradlew spotlessApply`
5. **Lint**: `./gradlew checkstyleMain pmdMain`

See [docs/ONBOARDING.md](docs/ONBOARDING.md) for full setup details.

## Test File Placement

### Java (Maven Standard Layout)

All Java tests live in the **mirror directory** to their source file.

```
platform/java/<module>/src/
  main/java/com/ghatana/...   ← production code
  test/java/com/ghatana/...   ← tests (same package, same relative path)
```

Example:
```
src/main/java/com/ghatana/platform/core/exception/ErrorResponseBuilder.java
src/test/java/com/ghatana/platform/core/exception/ErrorResponseBuilderTest.java
```

**Naming convention:** `<ClassName>Test.java` for unit tests;
`<ClassName>IT.java` for integration tests (Failsafe plugin picks these up).

**Async tests MUST** extend `EventloopTestBase` from `libs:activej-test-utils`.
See the [Async Patterns Guide](docs/platform-libraries/ASYNC_PATTERNS_GUIDE.md).

---

### TypeScript / React (Co-located `__tests__/`)

TypeScript tests live in **`__tests__/` subdirectories co-located with the
source they test**. This applies to all TypeScript packages under
`platform/typescript/`, `products/`, and `shared-services/`.

```
platform/typescript/design-system/src/
  atoms/
    Button.tsx
    __tests__/
      Button.test.tsx
  molecules/
    Alert.tsx
    __tests__/
      Alert.test.tsx
```

**Naming convention:** `<ComponentName>.test.tsx` (or `.test.ts` for
non-JSX). Snapshot files live in `__snapshots__/` inside the same
`__tests__/` directory (auto-generated by Jest).

**Why co-location?** TypeScript tests are statically typed and import their
subject directly by relative path. Co-location keeps the import short, makes
refactoring easier, and is consistent with Vitest/Jest discovery defaults.

**Do NOT** use a top-level `__tests__/` or `test/` folder for TypeScript —
this diverges from the convention and makes discovery unreliable.

---

### React Native (guardian / parent-mobile)

Follow the same co-located pattern as TypeScript above. See the detailed
guide at:
[products/dcmaar/apps/guardian/apps/parent-mobile/JEST_TESTING_GUIDE.md](products/dcmaar/apps/guardian/apps/parent-mobile/JEST_TESTING_GUIDE.md)

Key rule: **mock individual screens, not the Navigator**.

## Documentation Requirements

Every new public Java class must have all four `@doc.*` Javadoc tags:

```java
/**
 * @doc.type  [class|interface|record|enum]
 * @doc.purpose  One-line description
 * @doc.layer  [core|product|platform]
 * @doc.pattern  [Service|Repository|ValueObject|Exception|etc]
 */
```

New files missing these tags will **fail the `checkDocTags` check** in CI.
Existing violations are tracked in [gradle/doc-tag-baseline.txt](gradle/doc-tag-baseline.txt).

## Pull Request Checklist

- [ ] Tests added / updated
- [ ] `./gradlew spotlessApply` has been run
- [ ] `./gradlew checkstyleMain pmdMain` passes
- [ ] All public Java classes have required `@doc.*` tags
- [ ] `docs/` updated if architecture changed
- [ ] CHANGELOG updated in affected modules
